## Summary
CMU's comprehensive systems textbook covering the entire stack a programmer interacts with: data representation, machine-level programming (x86-64), processor architecture, memory hierarchy (caches, virtual memory), linking, exceptional control flow, system-level I/O, network programming, and concurrency. The canonical systems book.

## Key Takeaways
- Understanding the memory hierarchy (L1/L2/L3/RAM/disk) explains why some code is 100x faster than equivalent alternatives
- Virtual memory is the abstraction that makes modern multitasking possible — every process gets its own address space
- Linking is the invisible step between compilation and execution that most programmers never think about until it breaks
